# Figma 转 Onlook

将 Figma 设计直接导出到 Onlook。无需开发人员参与。本指南将向您展示如何构建设计、导出设计，并使用 Onlook 将其转换为可运行的 React 应用程序。

{/* ![Figma to Onlook Banner](/images/figma-to-onlook-banner.png) */}

*使用 Onlook 将您的 Figma 设计转化为可运行代码*

## 为何此集成改变一切

设计师不再止步于原型设计。借助 Onlook，您可以：

- 将 Figma 设计转换为清晰、结构化的 React 组件
- 使您的设计具有交互性和功能性
- 将您的 UI 连接到真实数据源
- 将您的作品部署为功能齐全的应用程序

## 逐步指南：从 Figma 到 React 应用

### 1. 构建您的 Figma 设计结构

为了顺利导入，您的 Figma 文件需要组织良好。以下是准备方法：

#### 使用自动布局

- 对所有父容器应用自动布局
- 使用适当的边距、间距和调整大小规则
- 为响应式布局设置水平/垂直调整大小

#### 清晰命名图层

- 使用描述性名称，如 `Header`、`CTA Button`、`Nav Bar`——而非 `Frame 23`
- 避免使用符号和嵌套混乱
- 按逻辑对相似组件进行分组

{/* ![Layer Naming](/images/layer-naming.png) */}

*正确命名的图层有助于生成更好的代码*

#### 构建可复用组件

- 使用一致的样式（字体、颜色等）
- 将重复元素转换为 Figma 组件

#### 设计系统思维

- 应用全局样式
- 使用共享组件库（如果可用）

### 2. 导入到 Onlook

1. 将 Figma 设计导出为 SVG 或 PNG
2. 打开 Onlook 并创建新项目
3. 导入设计资源
4. 使用 Onlook 工具将设计重新创建为 React 组件

### 3. 在 Onlook 中优化

这就是魔法发生的地方——将静态设计转化为真实可用的应用。

1. 使用 AI 提示改进组件
2. 实时预览
3. 添加交互和功能
4. 连接数据源（如果需要）

### 4. 部署应用

准备就绪时：

- 一键推送应用上线
- 与团队或用户分享
- 根据反馈迭代
