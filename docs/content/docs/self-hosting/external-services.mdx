# 外部服务

Onlook 需要多个外部服务来提供其功能。这些服务均可在 Apache 2.0 或同等宽松许可下自托管。

## 自托管外部服务

如需自托管这些服务的帮助，请通过 [founders@onlook.com](mailto:founders@onlook.com) 联系我们。

### 1. Supabase

- 对于可扩展部署，请遵循官方的 [Supabase 自托管指南](https://supabase.com/docs/guides/self-hosting/docker) 使用 Docker 进行部署。
- 对于单机部署，您可以通过 CLI 运行 supabase，参考[单机指南](/self-hosting/single-machine#3-start-backend-services)。

### 2. 沙箱提供商

- 对于可扩展部署，使用 E2B 的[自托管指南](https://github.com/e2b-dev/infra/blob/main/self-host.md) 将其部署到 GCP。
- 对于单机部署，我们未来将添加完全本地的沙箱提供商。

### 3. AI 提供商

配置自定义 AI 提供商：

1. 在 [`packages/ai/src/chat/providers.ts`](https://github.com/onlook-dev/onlook/blob/main/packages/ai/src/chat/providers.ts) 中更新 providers。我们已经支持 Anthropic 和 OpenRouter 作为示例。您可以按照相同的格式添加新的 provider。
2. 通过搜索 [`initModel`](https://github.com/search?q=repo%3Aonlook-dev%2Fonlook+%22await+initModel%22&type=code) 来更新使用情况。
3. 在 `apps/web/client/.env` 文件中将您的 API 密钥更新为 provider 所需的 API 密钥。

[!NOTE]
我们支持 [AI SDK providers](https://ai-sdk.dev/providers/ai-sdk-providers) 中的任何 provider。您可以通过以下 AI SDK provider 指南添加自定义 provider：[OpenAI compatible](https://ai-sdk.dev/providers/openai-compatible-providers/custom-providers) 和 [Community](https://ai-sdk.dev/providers/community-providers/custom-providers)。
