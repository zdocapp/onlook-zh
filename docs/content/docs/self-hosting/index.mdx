# 自托管 Onlook

Onlook 为需要完全控制其可视化开发环境的组织提供了灵活的自托管选项。选择最适合您基础设施和需求的部署方法。

## 部署选项

### 单机部署（推荐）

使用独立构建直接在单台服务器或虚拟机上部署 Onlook。这是自托管 Onlook 最简单且最高效的方式。

**最佳适用场景：**

- 中小型团队（5-50 名用户）
- 简单的生产部署
- 注重成本效益的组织
- 希望无需容器直接控制的团队

[查看单机部署指南 →](/self-hosting/single-machine)

### Docker Compose 方式

使用 Docker 容器运行 Onlook 以实现隔离部署。适用于容器化环境。

**最佳适用场景：**

- 已在使用 Docker 的组织
- 开发和预发布环境
- 需要容器隔离的团队
- 复杂的多服务部署

[查看 Docker Compose 指南 →](/self-hosting/docker-compose)

### 云部署（企业版）

将 Onlook 部署到云基础设施，具备适当的扩展性、监控和企业级功能。

**最佳适用场景：**

- 大型团队和企业部署
- 需要高可用性的组织
- 需要高级监控和合规性的团队
- 多区域部署

[查看云部署指南 →](/self-hosting/cloud-deployment)

## 架构概览

Onlook 由以下几个组件组成：

- **Web 客户端**：Next.js 应用程序
- **后端**：Supabase 用于数据库、认证和存储
