# 开发者文档与贡献指南

本节为开发者提供技术文档，帮助理解 Onlook 的架构、参与项目贡献或扩展其功能。

## 概述

Onlook 采用单体仓库（monorepo）结构，包含多个相互关联的应用和包，主要使用 Next.js、React、TypeScript 和 Vite 构建。

## 为 Onlook 做贡献

我们欢迎对 Onlook 的贡献！本指南将帮助您开始参与项目贡献。

在向此代码库贡献时，请先通过 [issues](https://github.com/onlook-dev/onlook/issues)、[Discord](https://discord.gg/hERDfFZCsH)、[电子邮件](mailto:contact@onlook.com) 或其他方式与代码库所有者讨论您希望进行的更改，然后再实施变更。

请注意我们有一份 [行为准则](https://github.com/onlook-dev/onlook/blob/main/CODE_OF_CONDUCT.md)，请在与项目的所有互动中遵守该准则。

### 本地运行项目

您可以使用[此处](/developers/running-locally)的说明从源代码构建项目。

### 拉取请求流程

1. 要创建拉取请求（PR），请先[创建项目分支](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo)。

![fork](/images/contribute-fork.png)

2. 在你的复刻中创建更改，并[从该复刻打开一个 PR。](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request-from-a-fork)

![contribute](/images/contribute-open-pr.png)

3. 使用更改的详细信息更新 PR 描述。如果相关，请链接 issue。

4. 务必勾选“允许维护者编辑”复选框。这允许维护者在必要时更新你的 PR，从而加快审查过程。[在此处查看更多信息](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork)。

![allow edits](/images/contribute-allow-edits.png)

5. 请求其中一位维护者进行审查。一旦被接受，他们将能够合并你的 PR。

### 风格指南

我们尝试遵循 [Clean Code](https://gist.github.com/wojteklu/73c6914cc446146b8b533c0988cf8d29) 中的指南和童子军规则：

“让代码比你发现它时更干净，而不是更混乱”。

### 测试

我们使用 [Bun](https://bun.sh/) 进行测试。

```bash
bun test
```

### 代码检查

我们使用 [ESLint](https://eslint.org/) 进行代码检查，使用 [Prettier](https://prettier.io/) 进行格式化。这会在每次提交时运行。

```bash
bun lint
bun format
```

### 类型安全

我们使用 [TypeScript](https://www.typescriptlang.org/) 来确保类型安全。

要检查 Web 客户端的类型，请运行：

```bash
cd apps/web/client && bun run typecheck
```
